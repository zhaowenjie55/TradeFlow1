# TradeFlow 黑客松 PPT 提纲

## 1. 封面
- 标题：TradeFlow
- 副标题：AI 驱动的跨境选品与套利分析工作台
- 一句话定位：把“海外爆款发现 -> 国内货源匹配 -> 利润测算 -> 风险判断”压缩成一个可执行的智能体流程。
- 视觉建议：使用工作台三栏界面作为主视觉，突出“分析任务”“候选商品”“结构化报告”三块。

演讲提示：
TradeFlow 不是一个单点工具，而是一条可运行的跨境选品链路。我们希望把原本分散在表格、1688、Amazon、翻译工具和经验判断里的流程，收敛成一个 AI 工作台。

## 2. 痛点
- 跨境卖家选品链路长，通常要在 Amazon、1688、翻译、汇率、物流、佣金之间来回切换。
- 爆款发现和货源匹配之间存在语言鸿沟，英文标题很难直接命中 1688 供货商品。
- 很多人只看售价，不会系统算清楚采购、国内运费、跨境物流、平台佣金和汇损。
- 风险判断高度依赖经验，侵权、竞争强度、供货稳定性很难结构化沉淀。

演讲提示：
真正难的不是“找到商品”，而是快速判断这是不是一个值得进入的套利机会。这个判断过程通常慢、碎片化，而且不可复用。

## 3. 方案
- TradeFlow 采用“两阶段 AI 分析”。
- 第一阶段：从海外平台发现候选爆款，输出候选列表。
- 第二阶段：用户选中一个候选后，系统自动完成中文改写、1688 语义召回、国内货源匹配、成本测算和最终报告生成。
- 输出不是一句建议，而是一份结构化报告，包含结论、利润率、风险等级、货源证据和推荐动作。

演讲提示：
我们刻意保留“用户选择候选”这个交互点。这样系统既保留 agent 自动化，也让人可以在关键决策点介入，适合黑客松 demo，也适合后续产品化。

## 4. 产品形态
- 前端是一个 Nuxt 4 + Pinia 的 AI 工作台。
- 左侧是任务参数，支持关键词、目标站点、利润率和数量配置。
- 中间展示海外爆款候选，用户可直接发起二次分析。
- 右侧展示思考日志和套利报告，能够看到模型推理链路、风险评估和成本拆解。
- 支持历史任务、报告列表、在线预览与 Markdown 下载。

演讲提示：
我们没有把结果藏在聊天框里，而是做成了一个工作台。因为跨境选品需要比对、筛选、切换和复盘，这种界面更适合真实业务。

## 5. 技术实现
- 前端：Nuxt 4 + Vue 3 + Pinia，支持桌面工作台和流式浏览两种界面模式。
- 后端：Spring Boot，采用异步任务编排，天然适配长耗时分析流程。
- 模型层：GLM 负责标题改写和报告叙事。
- 检索层：BGE-M3 + pgvector 做 1688 语义召回。
- 数据层：沉淀 Amazon 搜索结果、1688 商品快照、任务状态和最终报告，支持 fallback 和复盘。
- 系统设计重点：真实链路可用时走实时接口，不可用时自动降级到历史数据，确保 demo 稳定。

演讲提示：
这不是简单把 LLM 接上去。真正的工程价值在于我们把模型、向量检索、任务状态机和 fallback 机制串成了一条稳定链路。

## 6. Demo Case
- 示例关键词：Acrylic Desktop Organizer
- 海外候选示例：Amazon 售价 12.99 USD
- 国内匹配示例：1688 参考货源 15.00 CNY，国内运费 5.00 CNY
- 按当前仓库默认配置推导：
- 美元汇率：7.20
- 跨境物流费率：12%
- 平台佣金费率：15%
- 汇损费率：3%
- 推导结果：
- 目标售价约 93.53 CNY
- 总成本约 48.06 CNY
- 预估利润约 45.47 CNY
- 预估利润率约 48.6%

备注：
以上数字基于仓库中的 seed 商品价格、运费与 `application.yml` 的定价参数推导，不是口头估算。

演讲提示：
这一页是整场 demo 的抓手。我们不是只告诉用户“这个商品可能不错”，而是明确给出为什么值得做，以及利润空间来自哪里。

## 7. 差异化亮点
- 不是泛选品搜索，而是“发现 + 匹配 + 测算 + 风控”的完整闭环。
- 不是纯聊天，而是适合业务使用的工作台界面。
- 不是单次回答，而是可追踪的任务状态、日志链路和报告资产。
- 不是只依赖实时接口，而是通过历史快照和自动降级保证系统可演示、可迭代、可扩展。
- 后续可继续扩展更多海外站点、更多国内货源渠道和更丰富的风控信号。

演讲提示：
黑客松里很多项目停留在“能回答一个问题”。TradeFlow 更像是一个最小可行产品，而不是一个单点 demo。

## 9. 收尾页
- 结论：TradeFlow 让跨境选品从“经验驱动”走向“数据 + AI 驱动”。
- 结束语：从海外爆款发现，到国内货源和利润判断，TradeFlow 用一条可运行的智能体流程，把复杂选品决策压缩成几分钟。
